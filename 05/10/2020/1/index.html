The Simplified SMO Algorithm

Posted on 05-10-2020 In Machine Learning

From Stanford CS229

This document describes a simplified version of the Sequential Minimal Optimization (SMO) algorithm for training support vector machines.

Recap of the SVM Optimization Problem

Recall from the lecture notes that a support vector machine computes a linear classifier of the form

f(x)=wTx+b(1) f(x)=w^{T} x+b \tag{1} f(x)=wTx+b(1)

Since we want to apply this to a binary classification problem, we will ultimately predict y = 1 if f(x)≥0f(x)\ge0f(x)≥0 and y=−1y=−1y=−1 if f(x)<0f(x)<0f(x)<0, but for now we simply consider the function f(x)f(x)f(x). By looking at the dual problem as we did in Section 6 of the notes, we see that this can also be expressed using inner products as

f(x)=∑i=1mαiy(i)⟨x(i),x⟩+b(2) f(x)=\sum_{i=1}^{m} \alpha_{i} y^{(i)}\langle x^{(i)}, x\rangle+b \tag{2} f(x)=i=1∑m​αi​y(i)⟨x(i),x⟩+b(2)

where we can substitute a kernel K(x(i),x)K(x^{(i)}, x)K(x(i),x) in place of the inner product if we so desire.

The SMO algorithm gives an efficient way of solving the dual problem of the (regularized) support vector machine optimization problem, given in Section 8 of the notes. Specifically, we wish to solve:

The KKT conditions can be used to check for convergence to the optimal point. For this problem the KKT conditions are

αi=0⇒y(i)(wTx(i)+b)≥1(6) \alpha_{i}=0 \quad \Rightarrow \quad y^{(i)}\left(w^{T} x^{(i)}+b\right) \geq 1 \tag{6} αi​=0⇒y(i)(wTx(i)+b)≥1(6)

The Simplified SMO Algorithm

the SMO algorithm selects two α parameters, αi\alpha_iαi​ and αj\alpha_jαj​ and optimizes the objective value jointly for both these α\alphaα’s. Finally it adjusts the bbb parameter based on the new α\alphaα’s. This process is repeated until the α\alphaα’s converge. We now describe these three steps in greater detail.

Selecting α Parameters

Much of the full SMO algorithm is dedicated to heuristics for choosing which αi\alpha_iαi​ and αj\alpha_jαj​ to optimize so as to maximize the objective function as much as possible. For large data sets, this is critical for the speed of the algorithm, since there are m(m−1)m(m−1)m(m−1) possible choices for αi and αj\alpha_jαj​, and some will result in much less improvement than others.

However, for our simplified version of SMO, we employ a much simpler heuristic. We simply iterate over all αi,i=1,…,m\alpha_i, i = 1,\dots,mαi​,i=1,…,m. If αi\alpha_iαi​ does not fulfill the KKT conditions to within some numerical tolerance, we select αj\alpha_jαj​ at random from the remaining m−1m−1m−1 α\alphaα’s and attempt to jointly optimize αi\alpha_iαi​ and αj\alpha_jαj​. If none of the α\alphaα’s are changed after a few iteration over all the αi\alpha_iαi​’s, then the algorithm terminates. It is important to realize that by employing this simplification, the algorithm is no longer guaranteed to converge to the global optimum (since we are not attempting to optimize all possible αi\alpha_iαi​, αj\alpha_jαj​ pairs, there exists the possibility that some pair could be optimized which we do not consider).
